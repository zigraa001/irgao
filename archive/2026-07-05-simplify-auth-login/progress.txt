## Codebase Patterns
- JS bundle is auto-generated from js/app/*.js via `npm run build:js` (scripts/build-app-js.js); never edit js/app.bundle.js directly
- Build script auto-bumps the ?v= cache-buster on CSS/JS links in app.html based on content hash
- Typecheck: `npm run typecheck` (requires `npm install` first if node_modules missing)
- Auth card visibility managed by hideAllAuthCards() which hides all IDs in a hardcoded array; when removing a card, also remove its ID from that array and its error ID from the error array
- Frontend is plain vanilla JS with ~75 top-level let/const globals shared across features in a single concatenated scope
- auth-routes.js manually re-registers routes from mountRoleSignup — when changing role-signup.js handlers, also update auth-routes.js
- Passenger registration is now a single card (#register-passenger-card) with inline email OTP — no separate verify card
- Watch for smart/curly quotes (' ') in JS — they pass typecheck but crash at runtime; always use ASCII quotes
- Google button is hidden for non-passenger roles via applyPortalLabels; keep this in sync when modifying role switching
