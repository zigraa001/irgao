## Codebase Patterns
- Backend is plain Express 5 + raw MySQL via `mysql2` (src/db.js exports `query`, `queryOne`). No ORM.
- Route modules live in src/*-routes.js, each `express.Router()`, mounted in src/api.js under /api/<area>. `express.json()` is applied in server.js.
- Auth utilities in src/auth.js: `hashPassword`/`verifyPassword` (bcrypt, 10 rounds), `signToken`/`verifyToken` (HMAC token), middleware `requireAuth` (sets req.user={id,name,role}) and `requireRole(role)` factory. Compose as `requireAuth, requireRole('admin')`.
- Convention: every route that returns a user shapes it with a local `publicUser(u)` => {id,name,email,role} — NEVER return passwordHash. Emails are lowercased before insert/lookup. EMAIL_RE = /^[^\s@]+@[^\s@]+\.[^\s@]+$/, password min length 6.
- Insert pattern: `INSERT ...` returns `{ insertId }`, then re-read with `SELECT * FROM users WHERE id = ?` so the response reflects DB defaults/timestamps.
- Tests use node's built-in runner (`npm test` = `node --test`), no supertest. They are DB-FREE. To test a route without a real MySQL, inject an in-memory fake into `require.cache[require.resolve('../src/db')]` BEFORE requiring the route module (route modules destructure `{query,queryOne}` at load time), mount on a throwaway express app, and drive with real `fetch`. Auth is real: set process.env.AUTH_SECRET then use signToken to mint tokens. See test/admin-routes.test.js.
- Typecheck = `npm run typecheck` (tsc --noEmit, allowJs, checkJs:false, strict:false). Keep it green.

## Sun Jun 28 - US-001 Admin-only create-user API

Validates required name/email/password, EMAIL_RE, password>=6, and role in {operator,admin} (rejects 'customer'/unknown with 400). 409 on existing lowercased email. Stores bcrypt hash via hashPassword; returns 201 { user: {id,name,email,role} } (no passwordHash).
- Mounted the router at /api/admin in src/api.js.
- Added test/admin-routes.test.js (12 tests: 401/403 guards, all validation branches, happy path for operator+admin, passwordHash never leaked, 409 duplicate). Full suite: 29 pass. Typecheck clean.
- **Learnings for future iterations:**
  - The require.cache db-fake trick (see Codebase Patterns) is the way to integration-test admin/list/booking routes without MySQL. The fake matches on whitespace-collapsed SQL prefixes; keep route SQL predictable.
  - requireRole only checks `req.user.role !== role` for a SINGLE role string. There is no built-in "any of these roles" helper yet — add one if a route needs to allow multiple roles.
  - Pre-existing unrelated working-tree changes (scripts/seed.js, scripts/ralph/archive/, tasks/*.md) are harness/env artifacts — keep feature commits focused on src/ + test/ + PRD/progress.

- **Learnings for future iterations:**
  - The db-fake matches by SQL prefix; list query starts `SELECT id, name, email, role, createdAt FROM users` — distinct enough not to collide with the existing `WHERE email` / `WHERE id` branches. When adding new SQL, keep the leading shape unambiguous.
  - Filterable vs creatable roles differ on purpose: customers can be LISTED but never CREATED via /api/admin/users.
